# ADR-0003: edit_file 归一化匹配级联与 ground-truth 回带

- 状态：已接受（2026-07-18）
- 关联：DESIGN-P0.md §5；REVIEW-DESIGN-P0.md B2-2

## 背景

`edit_file` 的 `old` 参数要求与磁盘内容逐字节一致。即使协议走代码围栏（ADR-0002），空白、缩进（tab/space）、行尾（\r\n）失真仍会发生；匹配失败后模型凭记忆重试大概率再次失真，进入死循环烧掉 maxIterations。

## 决策

P0 内实现匹配级联与失败反馈：

1. 级联顺序：精确匹配 → 行首尾空白归一匹配 → 逐行 trim 后匹配；任何一级命中后**按磁盘原文**执行替换。
2. 守卫：任一级归一化后若匹配不唯一，立即报错（不猜测），错误信息说明"归一后多处命中"。
3. 全部失败时，错误信息回带磁盘上带行号的邻近真实片段（ground truth），让模型基于磁盘现状修正，而非凭记忆重试。

## 理由

- Claude Code 类工具的标准做法，成本约半天，直接消解 copilot-web 有损信道下 edit_file 的系统性失败模式。
- 回带磁盘片段把重试从"开环"变"闭环"，显著降低纠错轮次消耗。

## 后果

- edit_file 单测需覆盖各级命中、归一后不唯一、全失败回带片段等分支。
