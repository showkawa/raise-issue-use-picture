# ADR-0007: 账号风控保护——发送节奏与风险披露

- 状态：已接受（2026-07-18）
- 关联：DESIGN-P0.md §10、§13；REVIEW-DESIGN-P0.md H2

## 背景

自动化高频驱动 Copilot 网页版（单任务 20+ 轮、固定 1.5s 机器节奏）可能触发企业租户异常行为检测或违反 M365 使用条款，后果是用户账号被限制——这不是工程能修复的问题，只能降速 + 披露。

## 决策

1. `minSendIntervalMs` 默认提高到 ≥3000ms，并叠加随机抖动（避免机器节奏特征）。
2. 单任务轮次预算（maxIterations / maxTurnsPerConversation）默认值同时按"账号保护"口径设定，不单纯按任务成功率。
3. 风险写入 §13，README 与首次运行时向用户披露"自动化使用可能违反 M365 条款/触发风控，账号风险自负"。
4. 用户可在 config 调低间隔，文档注明风险自负。

## 后果

- 单任务耗时近似翻倍（约多等 1 分钟/30 轮），换取账号被封尾部风险的显著降低。
