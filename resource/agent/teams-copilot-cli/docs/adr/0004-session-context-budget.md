# ADR-0004: 会话级上下文预算账本、协议遗忘检测与动态回灌收缩

- 状态：已接受（2026-07-18）
- 关联：DESIGN-P0.md §3、§4.4、§7、§10；REVIEW-DESIGN-P0.md B3

## 背景

分片与轮次上限只解决"单条消息发得出去"，未回答"会话总上下文装不装得下"。Copilot 网页版窗口大小未知且不可控，长任务中最早注入的协议提示词会被挤出窗口，模型转为自由文本输出，loop 只能按 malformed 纠错空转。§7 的重建路径只覆盖 `healthy()===false`，不覆盖"会话活着但协议已被遗忘"。

## 决策

P0 内实现三项：

1. **每会话字符预算账本**：累计双向字符数，逼近阈值（配置化，保守默认 40k）时主动 rotate session，复用 §7 的"重建 + 协议重发 + 进度摘要"路径。默认值在真机协议冒烟（ADR-0002）时标定。
2. **协议遗忘检测**：连续 2 次 malformed 且回复呈自由问答形态时，视同协议丢失，直接走 rotate-session 路径而非继续纠错。
3. **动态回灌收缩**：RESULT 截断偏激进——测试输出只保留失败摘要 + 尾部 N 行；回灌预算按剩余会话预算动态收缩。

## 理由

- 三者本质是计数器 + 分类判定 + 截断策略，实现成本低，且全部复用已设计的 rotate 路径。
- 不做总量模型，长任务"中途失忆"是必然而非偶然。

## 后果

- rotate 成为常态路径而非异常路径，进度摘要的质量直接影响长任务成功率（P1 的智能 compaction 优先级上升）。
- MockProvider 集成测试需覆盖：预算触发 rotate、遗忘检测触发 rotate。
