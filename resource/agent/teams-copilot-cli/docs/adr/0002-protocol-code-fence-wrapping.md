# ADR-0002: 协议默认走代码围栏包裹，真机协议冒烟提前

- 状态：已接受（2026-07-18）
- 关联：DESIGN-P0.md §4.2、§12、§13；REVIEW-DESIGN-P0.md B2

## 背景

agent 回复要经过"模型生成 → Copilot 渲染 → SignalR/DOM 捕获 → sanitizeMarkdown"四道变换。裸的 `<<<TOOL ...>>>` 自定义围栏在此链路上可能遭遇智能引号、全角化、空白/换行归一等失真，导致 JSON 解析失败与 `edit_file` 逐字节匹配系统性失败。该格式决策影响 protocol.ts、system-prompt.ts、纠错逻辑三处，晚改成本高。

## 决策

1. 默认策略：要求模型把整个协议回复放入 ``` 代码围栏，`<<<TOOL>>>`/`<<<DONE>>>` 自定义围栏在代码围栏内部使用。
2. 真机协议冒烟（握手块往返保真校验）从实施步骤 7 提前到步骤 2 之后，用真机数据定案格式后再进入后续步骤。
3. 终端展示层负责剥掉外层代码围栏后再渲染 commentary。

## 理由

- 代码围栏是 Copilot 渲染管线中最不易被美化/转义的区域，等于给协议加一层免费信道保护。
- 协议格式是后续所有步骤的地基；把最大未知数（格式遵循度、信道保真）放在最后一步等于把最大风险留到最晚发现。

## 后果

- 协议提示词与解析器需处理"外层代码围栏 + 内层自定义围栏"的双层结构，包括围栏未闭合（回复被截断）的判定。
- 实施顺序调整：步骤 2 之后插入真机冒烟节点。
