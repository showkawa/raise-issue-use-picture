# {project_name} - 产品需求文档

请以资深产品经理的身份，为 {project_name} 输出一份结构完整的 PRD。

## 要求

1. **产品概述**：一句话描述产品定位与核心价值
2. **目标用户**：2-3 个用户画像
3. **核心功能**：按优先级排列（P0 / P1 / P2）
4. **用户故事**：每个核心功能至少 1 个用户故事
5. **验收标准**：每个功能对应的可量化标准
6. **非功能需求**：性能、安全、可用性指标
7. **里程碑**：4 个阶段的时间线

请使用符合行业规范的 PRD 文档格式，直接输出 Markdown，不要有任何前缀寒暄或后缀追问。
